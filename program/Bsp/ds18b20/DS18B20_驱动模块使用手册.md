# DS18B20 温度传感器驱动模块使用手册

## 1. 驱动概述

DS18B20 驱动是基于单总线（1-Wire）协议的温度传感器驱动模块，通过 GPIO 模拟单总线时序实现温度采集。驱动内部使用 `delay_us` 公共微秒延时服务和 FreeRTOS 临界区保护，确保单总线时序的准确性。

| 项目 | 说明 |
|------|------|
| 文件 | `ds18B20.c`、`ds18B20.h` |
| 依赖 | `delay_us` 微秒延时模块、FreeRTOS（临界区保护） |
| 通信方式 | 单总线（1-Wire），GPIO 模拟 |
| 测量范围 | -55°C ~ +125°C |
| 分辨率 | 0.0625°C（12 位精度） |
| 数据引脚 | `DATA_GPIO_Port` / `DATA_Pin`（由 CubeMX 在 `main.h` 中定义） |

### 功能特性

- **单总线通信**：通过 GPIO 模拟 1-Wire 协议，无需额外硬件外设
- **12 位精度**：默认配置为 12 位分辨率，温度精度 0.0625°C
- **临界区保护**：使用 FreeRTOS `taskENTER_CRITICAL`/`taskEXIT_CRITICAL` 保护单总线时序，防止任务切换打断时序
- **浮点输出**：`ds18b20_read()` 直接返回 `float` 类型温度值（摄氏度）
- **公共延时依赖**：使用 `delay_us` 模块提供的 DWT 微秒延时，避免重复初始化

## 2. 模块架构

```
┌──────────────────────────────────────────────┐
│                用户应用层                    │
│         ds18b20_init() / ds18b20_read()      │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│              DS18B20 驱动层                  │
│  ┌────────────┐  ┌────────────┐              │
│  │ 1-Wire时序  │  │ 温度解析   │              │
│  │ Reset      │  │ 原始数据   │              │
│  │ WriteByte  │  │ → float    │              │
│  │ ReadByte   │  │            │              │
│  └─────┬──────┘  └────────────┘              │
│        │                                     │
│  taskENTER_CRITICAL() / taskEXIT_CRITICAL()  │
│  (保护时序临界区)                            │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│           delay_us / HAL GPIO                │
│    delay_us()    HAL_GPIO_WritePin/ReadPin   │
└──────────────────────────────────────────────┘
```

### 单总线时序流程

```
ds18b20_read() 调用流程:
    │
    ├── 1. DS18B20StartConvert()  发送温度转换命令
    │      Reset → SkipROM → StartConvert
    │
    ├── 2. ResetDS18B20()         复位总线，等待传感器应答
    │      拉低 500us → 释放 → 等待应答脉冲
    │
    ├── 3. WriteByte(SkipROM)     跳过 ROM 地址（单传感器）
    ├── 4. WriteByte(ReadScratchpad)  读暂存器
    │
    ├── 5. ReadByte() × 2         读取温度低字节 DL 和高字节 DH
    │
    └── 6. 温度计算: (DH<<8 | DL) × 0.0625 = 温度值 (°C)
```

## 3. 核心定义

### ROM 命令

| 宏 | 值 | 说明 |
|----|-----|------|
| `SkipROM` | 0xCC | 跳过 ROM（单传感器场景，无需匹配地址） |
| `SearchROM` | 0xF0 | 搜索 ROM |
| `ReadROM` | 0x33 | 读 ROM |
| `MatchROM` | 0x55 | 匹配 ROM |
| `AlarmROM` | 0xEC | 报警 ROM |

### 功能命令

| 宏 | 值 | 说明 |
|----|-----|------|
| `StartConvert` | 0x44 | 启动温度转换 |
| `ReadScratchpad` | 0xBE | 读暂存器（9 字节） |
| `WriteScratchpad` | 0x4E | 写暂存器（TH、TL、精度） |
| `CopyScratchpad` | 0x48 | 复制暂存器到 EEPROM |
| `RecallEEPROM` | 0xB8 | 从 EEPROM 恢复到暂存器 |
| `ReadPower` | 0xB4 | 读供电方式 |

### 精度配置

| 精度寄存器值 | 分辨率 | 转换时间 |
|-------------|--------|---------|
| 0x1F | 9 位 | 93.75 ms |
| 0x3F | 10 位 | 187.5 ms |
| 0x5F | 11 位 | 375 ms |
| 0x7F | 12 位（默认） | 750 ms |

| 函数 | 说明 |
|------|------|
| `void ds18b20_init(void)` | 初始化 DS18B20：初始化 delay_us、配置 GPIO、复位传感器、设置精度（12 位）、启动首次转换 |
| `float ds18b20_read(void)` | 读取温度值，返回摄氏度浮点数 |

## 5. 使用示例

### 基本使用

```c
#include "ds18B20.h"
#include <stdio.h>

void TempTask(void *arg)
{
    float temperature;

    /* 初始化 DS18B20 温度传感器 */
    ds18b20_init();

    for (;;)
    {
        /* 读取温度 */
        temperature = ds18b20_read();

        /* 打印温度 */
        printf("Temperature: %.2f C\r\n", temperature);

        /* 间隔 500ms 读取一次 */
        osDelay(500);
    }
}
```

### 搭配 UartDrv 调试输出

```c
#include "ds18B20.h"
#include "uart_drv.h"
#include "usart.h"
#include <stdio.h>

static UartDrv_t g_uart1Drv;

void TempTask(void *arg)
{
    /* 初始化串口驱动并设置为 printf 调试输出口 */
    UartDrv_Init(&g_uart1Drv, &huart1);
    UartDrv_SetDebugPort(&g_uart1Drv);

    /* 初始化 DS18B20 */
    ds18b20_init();
    printf("DS18B20 Demo Started\r\n");

    float temp;
    for (;;)
    {
        temp = ds18b20_read();
        printf("Temp: %.2f C\r\n", temp);
        osDelay(1000);
    }
}
```

## 6. NFCAttend 工程实际应用

DS18B20 模块在 NFCAttend 工程中未直接使用，但在 Demo_TempSensor 示例工程中展示了完整用法。以下为 Demo_TempSensor 中的关键代码（freertos.c → StartTempTask）：

```c
#include "ds18B20.h"
#include "uart_drv.h"
#include "usart.h"
#include <stdio.h>

static UartDrv_t g_uart1Drv;

void StartTempTask(void *argument)
{
    float temperature;

    /* 初始化 DS18B20 温度传感器 */
    ds18b20_init();

    /* 初始化串口驱动并设置为 printf 调试输出口 */
    UartDrv_Init(&g_uart1Drv, &huart1);
    UartDrv_SetDebugPort(&g_uart1Drv);

    /* Infinite loop */
    for(;;)
    {
        /* 读取温度 */
        temperature = ds18b20_read();

        /* 通过 printf 打印温度数据 (已重定向到 USART1) */
        printf("Temperature: %.2f C\r\n", temperature);

        /* 等待 100ms */
        osDelay(100);
    }
}
```

> **设计要点**：`ds18b20_read()` 内部会启动温度转换并立即读取，首次调用可能返回上一次转换的结果。建议初始化后等待 750ms（12 位精度转换时间）再读取，或忽略首次读数。

## 7. 注意事项

- **GPIO 配置**：数据引脚 `DATA_Pin` 需在 CubeMX 中定义为输出模式（推挽），驱动内部通过 `HAL_GPIO_WritePin`/`HAL_GPIO_ReadPin` 控制。引脚宏定义在 `main.h` 中。
- **上拉电阻**：单总线需要外接 4.7kΩ 上拉电阻到 VCC，否则通信不稳定。
- **临界区保护**：驱动内部使用 FreeRTOS 临界区（`taskENTER_CRITICAL`/`taskEXIT_CRITICAL`）保护单总线时序。在非 FreeRTOS 环境中使用需自行替换为关中断操作。
- **延时依赖**：驱动依赖 `delay_us` 模块提供的 DWT 微秒延时，`ds18b20_init()` 内部会调用 `delay_us_init()` 初始化。
- **转换时间**：12 位精度下温度转换需要 750ms。`ds18b20_read()` 每次调用都会启动新转换并读取上次结果，建议读取间隔不小于 750ms。若需更快响应，可降低精度配置（修改 `DS_PRECISION` 宏）。
- **多传感器**：当前驱动仅支持单传感器场景（使用 `SkipROM` 命令）。如需多传感器，需实现 `MatchROM` 地址匹配逻辑。
- **任务优先级**：由于驱动使用临界区，调用 `ds18b20_read()` 时会短暂禁用任务切换（约 1ms），高优先级任务需注意此延迟。
