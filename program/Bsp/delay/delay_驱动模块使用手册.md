# delay 公共微秒延时驱动使用手册

## 1. 驱动概述

`delay_us` 是一个基于 Cortex-M 内核 **DWT（Data Watchpoint and Trace）CYCCNT 周期计数器** 的公共微秒级延时服务模块。所有需要精确微秒延时的驱动（如 DS18B20、RC522 等）统一通过此模块获取延时能力，避免各驱动各自初始化 DWT 带来的冗余和潜在冲突。

| 项目 | 说明 |
|------|------|
| 文件 | `delay_us.c`、`delay_us.h` |
| 依赖 | Cortex-M DWT 寄存器、`SystemCoreClock`（由 `main.h` 提供） |
| 精度 | 1 微秒级（取决于系统主频，如 168MHz 时精度约 6ns） |

---

## 2. 模块架构

```
┌───────────────────────────────────────┐
│     各驱动 (DS18B20 / RC522 / ...)    │
│            调用 delay_us()             │
└─────────────────┬─────────────────────┘
                  │
┌─────────────────▼─────────────────────┐
│         delay_us 公共延时层            │
│  ┌──────────────┐  ┌──────────────┐   │
│  │ s_dwt_inited │  │ DWT->CYCCNT  │   │
│  │ 初始化标志    │  │ 周期计数器    │   │
│  └──────────────┘  └──────────────┘   │
└─────────────────┬─────────────────────┘
                  │ 读取 DWT 寄存器
┌─────────────────▼─────────────────────┐
│        Cortex-M 内核 DWT 硬件          │
└───────────────────────────────────────┘
```

**核心原理**：DWT 的 `CYCCNT` 寄存器每个时钟周期自增 1。延时函数记录起始计数值，通过计算 `cycles = (SystemCoreClock / 1000000) * us` 得到目标周期数，轮询等待差值达到目标即可。

---

## 3. 核心数据结构

本模块无导出数据结构，仅内部使用一个静态标志：

```c
static volatile uint8_t s_dwt_inited = 0;  // DWT 初始化标志（0=未初始化）
```

---

## 4. API 参考

| 函数 | 说明 |
|------|------|
| `void delay_us_init(void)` | 初始化 DWT 微秒延时。多次调用安全（仅首次实际初始化）。需在系统启动时调用一次 |
| `void delay_us(uint32_t us)` | 微秒级阻塞延时。可重入，多任务/中断中均可调用 |

---

## 5. 使用示例

### 基本使用

```c
#include "delay_us.h"

/* 系统初始化时调用一次 */
delay_us_init();

/* 在任意位置使用微秒延时 */
delay_us(500);   // 延时 500 微秒
delay_us(10);    // 延时 10 微秒
```

### 在其他驱动中使用（以 DS18B20 为例）

```c
#include "delay_us.h"

void ds18b20_init(void)
{
    delay_us_init();          // 初始化微秒延时（安全，可重复调用）
    /* ... 后续使用 delay_us() 完成单总线时序 ... */
}
```

## 7. NFCAttend 工程实际应用

NFCAttend 考勤系统中，`delay_us` 模块由 RC522 驱动（NFC 刷卡）和 DS18B20 驱动（温度传感器）间接使用。

### RC522 软件 SPI 时序保护

在 `rc522.c` 的 `spi_transfer_byte()` 函数中，`delay_us` 提供微秒级延时控制 SCK 边沿：

```c
/* 软件 SPI 时序: ~1 µs 边沿间隔 */
static uint8_t spi_transfer_byte(uint8_t data)
{
    /* ... 使用 delay_us(1) 控制 SCK 脉冲宽度 ... */
}
```

> RC522 的 `RC522_Platform_Init()` 内部会调用 `delay_us_init()` 初始化 DWT 定时器。由于 `delay_us_init()` 可重复调用（第二次起不重置计数器），DS18B20 再调用时不会产生冲突。

---

## 6. 注意事项

- **初始化时机**：`delay_us_init()` 需在系统时钟配置完成后调用，因为延时计算依赖 `SystemCoreClock` 的正确值。
- **重复调用安全**：内部使用 `s_dwt_inited` 标志保护，多次调用不会重置计数器。
- **多任务安全**：`delay_us()` 仅读取 `CYCCNT`（只读不写），多任务/多驱动共用不会冲突。
- **阻塞特性**：`delay_us()` 是忙等待（busy-wait）阻塞延时，CPU 在此期间不释放。在 FreeRTOS 任务中如需长延时，应使用 `osDelay()` 释放 CPU。
- **DWT 可用性**：DWT 在 Cortex-M3/M4/M7 内核上可用。若使用 M0/M0+ 内核（无 DWT），需替换为定时器方案。
