# RC522 / MFRC522 RFID 读卡器驱动使用说明

## 概述

本驱动为 MFRC522/RC522 射频读写芯片提供了一套 **高内聚、低耦合** 的驱动程序。通过平台抽象接口 (`RC522_IO_t`) 与硬件彻底解耦，可以方便地移植到任意嵌入式平台。

### 设计架构

```
┌────────────────────────────────────────────────┐
│              应用层 (Application)                │
│  (freertos.c / 其他业务代码)                    │
├────────────────────────────────────────────────┤
│           rc522.h / rc522.c                      │
│          核心驱动层 (纯逻辑，无硬件依赖)            │
│  ┌──────────────────────────────────────────┐   │
│  │  高级 API: ScanCard, ReadAllSectors, ... │   │
│  │  基础 API: Request, Anticoll, Auth, ...  │   │
│  │  寄存器层: write_reg, read_reg           │   │
│  └──────────────┬───────────────────────────┘   │
├─────────────────┼────────────────────────────────┤
│   rc522_platform_stm32.c  (平台适配层)            │
│   ┌─────────────┴──────────────┐                │
│   │  CS/RST 控制  │  SPI传输   │  延时          │
│   │  (GPIO写)     │  (Bitbang)  │  (DWT/HAL)   │
│   └────────────────────────────┘                │
├────────────────────────────────────────────────┤
│            STM32 HAL / 硬件层                    │
│          (GPIO, TIM, CoreDebug)                 │
└────────────────────────────────────────────────┘
```

### 文件结构

| 文件 | 说明 | 是否可移植 |
|------|------|-----------|
| `rc522.h` | 头文件：寄存器定义、API 声明、平台接口定义 | **独立，无平台依赖** |
| `rc522.c` | 核心实现：所有 RC522 操作逻辑 | **独立，无平台依赖** |
| `rc522_platform_stm32.c` | STM32 平台适配：SPI 模拟、DWT 延时 | 需根据平台修改 |

## 核心数据结构

### RC522_IO_t — 平台抽象接口

驱动通过此接口与硬件平台解耦，用户需实现全部 5 个回调函数：

```c
typedef struct {
    void    (*cs_control)(uint8_t level);    // 片选控制 (0=选中, 1=释放)
    void    (*rst_control)(uint8_t level);   // 复位控制 (0=复位, 1=正常)
    uint8_t (*spi_transfer)(uint8_t data);   // SPI 全双工字节传输 (Mode 0)
    void    (*delay_us)(uint32_t us);        // 微秒延时
    void    (*delay_ms)(uint32_t ms);        // 毫秒延时
} RC522_IO_t;
```

### 状态码

| 宏 | 值 | 说明 |
|----|----|------|
| `RC522_OK` | 0 | 操作成功 |
| `RC522_ERR` | -1 | 操作失败 |
| `RC522_ERR_NO_TAG` | -2 | 无卡响应 |
| `RC522_ERR_COLLISION` | -3 | 防碰撞失败 |

### 卡片常量

| 宏 | 值 | 说明 |
|----|----|------|
| `RC522_PICC_REQIDL` | 0x26 | 寻空闲卡 |
| `RC522_PICC_REQALL` | 0x52 | 寻所有卡 |
| `RC522_PICC_AUTHENT1A` | 0x60 | 验证 A 密钥 |
| `RC522_PICC_AUTHENT1B` | 0x61 | 验证 B 密钥 |

### Mifare S50 卡片内存结构

Mifare S50 卡片有 16 个扇区，每个扇区 4 个块，每块 16 字节：

```
扇区 0: [块 0 (厂商块)] [块 1] [块 2] [块 3 (密钥A | 存取控制 | 密钥B)]
扇区 1: [块 4] [块 5] [块 6] [块 7 (密钥)]
...
扇区 15:[块 60][块 61][块 62][块 63 (密钥)]
```

- **块 0 (扇区 0)**：存储厂商数据 (UID 等)，出厂锁定不可写
- **扇区第 3 块**：存储密钥 A (6B) + 存取控制 (4B) + 密钥 B (6B)
- **默认出厂密钥**：`0xFF 0xFF 0xFF 0xFF 0xFF 0xFF`

## 快速开始 (STM32 平台)

### 1. CubeMX 引脚配置

在 STM32CubeMX 中配置以下 GPIO 引脚：

| 引脚 | 方向 | 说明 |
|------|------|------|
| NFC_NSS (PB15) | 推挽输出 | 片选 (低电平有效) |
| NFC_RST (PB14) | 推挽输出 | 复位 (高电平正常) |
| NFC_MOSI (PA0) | 推挽输出 | SPI 主机输出 |
| NFC_MISO (PB13) | 浮空输入 | SPI 主机输入 |
| NFC_SCK (PD9) | 推挽输出 | SPI 时钟 |
| NFC_GND (PB10) | 推挽输出 | 模块接地 (输出低电平) |

> 注：具体引脚号以实际工程中的 `main.h` 定义为准。

### 2. 添加驱动文件到工程

将以下文件添加到 MDK 工程：

- `Drivers/NFC/rc522.h`
- `Drivers/NFC/rc522.c`
- `Drivers/NFC/rc522_platform_stm32.c`

### 3. 初始化驱动

```c
#include "rc522.h"

void my_nfc_task(void)
{
    // 一键初始化 (含 DWT 延时初始化 + RC522 芯片初始化)
    RC522_Platform_Init();
    
    // 配置 ISO14443A 并开启天线
    RC522_ConfigISOType('A');
    
    // ... 后续操作
}
```

### 4. 读取卡片 UID

```c
uint8_t card_id[4];
char status = RC522_ScanCard(card_id);

if (status == RC522_OK) {
    printf("卡号: %02X%02X%02X%02X\n", 
           card_id[0], card_id[1], card_id[2], card_id[3]);
}
```

### 5. 读取卡片数据

```c
uint8_t default_key[6] = {0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF};
uint8_t card_id[4];

// 寻卡 + 防碰撞 + 选卡
if (RC522_ScanCard(card_id) == RC522_OK) {
    // 验证第 0 扇区密码
    if (RC522_AuthState(RC522_PICC_AUTHENT1A, 3, default_key, card_id) == RC522_OK) {
        uint8_t buf[16];
        // 读取第 0 扇区第 1 块数据
        if (RC522_ReadBlock(0, 1, buf) == RC522_OK) {
            // 处理 buf 中的数据
        }
    }
}
```

### 6. 写入卡片数据

```c
uint8_t default_key[6] = {0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF};
uint8_t card_id[4];
uint8_t data[16] = {0};

if (RC522_ScanCard(card_id) == RC522_OK) {
    if (RC522_AuthState(RC522_PICC_AUTHENT1A, 3, default_key, card_id) == RC522_OK) {
        // 将要写入的数据填充到 data
        RC522_WriteBlock(0, 1, data);  // 写入扇区0块1
    }
}
```

## 平台移植指南

要将驱动移植到非 STM32 平台，只需重新实现 `RC522_IO_t` 接口：

```c
/* 1. 定义平台 IO 回调函数 */
static void my_cs_control(uint8_t level) {
    if (level) GPIO_SetHigh(CS_PIN); else GPIO_SetLow(CS_PIN);
}

static void my_rst_control(uint8_t level) {
    if (level) GPIO_SetHigh(RST_PIN); else GPIO_SetLow(RST_PIN);
}

static uint8_t my_spi_transfer(uint8_t data) {
    // 软件 SPI Mode 0: CPOL=0, CPHA=0
    // 循环 8 位, MSB 优先
    for (int i = 7; i >= 0; i--) {
        SCK_LOW();
        if (data & 0x80) MOSI_HIGH(); else MOSI_LOW();
        SCK_HIGH();
        data <<= 1;
        if (MISO_READ()) data |= 1;
        SCK_LOW();
    }
    return data;
}

static void my_delay_us(uint32_t us) {
    // 实现微秒延时 (如 DWT, TIM, 空循环等)
}

static void my_delay_ms(uint32_t ms) {
    // 实现毫秒延时
}

/* 2. 初始化 RC522_IO_t 结构体 */
static const RC522_IO_t io = {
    .cs_control   = my_cs_control,
    .rst_control  = my_rst_control,
    .spi_transfer = my_spi_transfer,
    .delay_us     = my_delay_us,
    .delay_ms     = my_delay_ms
};

/* 3. 传入 IO 接口并初始化 RC522 */
RC522_Init((RC522_IO_t *)&io);
```

### SPI 时序要求

驱动使用 SPI Mode 0 (CPOL=0, CPHA=0) 通讯：

```
SCK   ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
      │  │  │  │  │  │  │  │  │  │  │  │
      ┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──
MOSI  ──┐  ┌────┐  ┌────┐  ┌────┐  ┌────
        │  │    │  │    │  │    │  │
MISO  ───┘  └────┘  └────┘  └────┘  └────
       ^上升沿采样  ^上升沿采样  ^上升沿采样
```

## API 参考

### 初始化与配置

| 函数 | 说明 |
|------|------|
| `RC522_Init(RC522_IO_t *io)` | 初始化 RC522 芯片，传递平台 IO 接口 |
| `RC522_Reset(void)` | 软复位 RC522 芯片 |
| `RC522_ConfigISOType(char type)` | 配置通讯协议 (当前只支持 `'A'`) |
| `RC522_AntennaOn(void)` | 开启天线 |
| `RC522_AntennaOff(void)` | 关闭天线 |
| `RC522_Platform_Init(void)` | STM32 平台一键初始化 (DWT + GND + RC522) |

### 卡片基本操作

| 函数 | 说明 |
|------|------|
| `RC522_Request(req_code, pTagType)` | 寻卡，获取卡片类型 |
| `RC522_Anticoll(pSnr)` | 防碰撞，获取卡片 UID (4 字节) |
| `RC522_Select(pSnr)` | 选卡，使后续操作针对此卡 |
| `RC522_AuthState(mode, addr, pKey, pSnr)` | 验证扇区密码 |
| `RC522_Read(addr, pData)` | 读指定地址块 (16 字节) |
| `RC522_Write(addr, pData)` | 写指定地址块 (16 字节) |
| `RC522_Halt(void)` | 使卡进入休眠状态 |
| `RC522_WaitCardOff(void)` | 阻塞等待卡离开 |

### 高级数据操作

| 函数 | 说明 |
|------|------|
| `RC522_ScanCard(cn)` | 一键寻卡: Request + Anticoll + Select |
| `RC522_ReadBlock(sec, blk, buf)` | 按扇区+块号读取 |
| `RC522_WriteBlock(sec, blk, buf)` | 按扇区+块号写入 |
| `RC522_ReadSector(sec, buf)` | 读取整个扇区 (4块 x 16字节) |
| `RC522_WriteSector(sec, buf)` | 写入扇区前 3 块 (密钥区不写) |
| `RC522_ReadAllSectors(data)` | 读取整张卡 (所有扇区) |
| `RC522_WriteAllSectors(data)` | 写入整张卡 (跳过厂商块和密钥区) |

## 注意事项

1. **时序敏感**: RC522 的 SPI 通讯频率建议不超过 10MHz。软件模拟 SPI 时，`delay_us(1)` 的延时已足够。
2. **天线控制**: 每次通讯完成后，天线可能会自动关闭。如果连续操作失败，请确保天线已开启。
3. **密码验证**: 每次读取或写入某个扇区前，必须先验证该扇区的密码。跨扇区操作需要重新验证。
4. **电源稳定**: RC522 模块工作时电流约 30mA，确保电源供应稳定，否则可能导致通讯不稳定。

## RTOS 多任务注意事项

### 时序保护（临界区）

本驱动在 `spi_transfer_byte()` 函数内部使用 `taskENTER_CRITICAL()` / `taskEXIT_CRITICAL()` 对 SPI 时序进行保护：

```
传输 1 字节 ≈ 32 µs  →  临界区长度 ≈ 32 µs（可接受）
```

优点：
- ✅ 粒度最细（只在字节级），不阻塞其他任务的中断响应太久
- ✅ 调用方无需关心临界区管理
- ✅ 多任务中 RC522 操作不会互相干扰

> **背景**：软件模拟 SPI 的每个 SCK 边沿之间仅 ~1 µs。在裸机或 FreeRTOS 中，如果 `spi_transfer_byte()` 执行期间被更高优先级的任务或中断抢占，SCK 脉冲宽度可能从 1 µs 被拉长到数毫秒，导致 MFRC522 误判数据。临界区保证了字节传输的原子性。

> **与 DS18B20 共用 DWT**：本驱动的 `delay_us()` 和 DS18B20 驱动的 `delay_us()` 统一使用 `Drivers/delay/delay_us.c` 提供的公共微秒延时服务。该服务基于 Cortex-M DWT CYCCNT 寄存器实现，只读不写，多任务/多驱动共用**不会冲突**。

## NFCAttend 工程实际应用

NFCAttend 考勤系统使用 RC522 读取 Mifare S50 卡片数据，流程如下：

### 卡片数据格式

| 扇区 | 块 | 内容 |
|------|-----|------|
| 0 | 1 | 账户头：工号(5B)、点数(4B)、卡类型(1B) + 校验和 |
| 1~8 | 4-35 | 头像图像数据（仅图像卡） |
| 9~15 | 36-61 | 姓名图像(扇区9~11)、部门图像(扇区12~15) |

### NFCAttend 中的 NFC 初始化（nfc_app.c → NFC_App_Init）

```c
void NFC_App_Init(void)
{
    /* 初始化蜂鸣器 */
    Beep_Init();

    /* 初始化 RC522 */
    RC522_Platform_Init();   // 内部初始化软件 SPI GPIO 和 DWT
    RC522_ConfigISOType('A');
    RC522_AntennaOn();

    /* 清空状态 */
    s_last_uid = 0;
    s_old_cid = 0;
    s_card_present = 0;
}
```

### NFCAttend 中的卡片读取流程（nfc_app.c → NFC_ReadCard）

```c
int NFC_ReadCard(CardInfo_t *pInfo, uint8_t *snr)
{
    /* 1. 稳定读出账户头 (block 1)，获取卡类型 */
    for (int retry = 0; retry < 3; retry++) {
        status = RC522_ScanCard(cardID);     // 寻卡
        status = RC522_AuthState(...);        // 密钥认证
        status = RC522_ReadBlock(0, 1, ...);  // 读块数据
    }

    /* 2. 根据卡类型读取所有扇区数据 */
    for (int sec = 0; sec <= max_sector; sec++) {
        for (int blk = 0; blk < 4; blk++) {
            /* 失败时重新 Halt + ScanCard + 认证 */
            status = RC522_ScanCard(cardID);
            status = RC522_AuthState(...);
            status = RC522_ReadBlock(sec, blk, pInfo->raw_data[block_idx]);
        }
    }

    RC522_Halt();
    return 0;
}
```

### NFCAttend 中的周期性寻卡（nfc_app.c → NFC_App_Update）

```c
void NFC_App_Update(void)
{
    /* 已有卡: 用 ScanCard 检测卡是否还在 */
    if (s_card_present) {
        status = RC522_ScanCard(snr);
        if (status != RC522_OK) {
            /* 连续 N 次检测失败才判离卡，防止偶发误检 */
            if (++s_miss_cnt >= CARD_MISS_THRESHOLD) {
                s_card_present = 0;
                s_callback(NFC_EVT_CARD_LEAVE, NULL);
            }
        }
        return;
    }

    /* 无卡: 主动寻卡 */
    status = RC522_ScanCard(snr);
    if (status == RC522_OK) {
        s_card_present = 1;
        NFC_ReadCard(&card_info, snr);
        /* 数据校验 + 防重复检测 + 声光反馈 */
    }

    RC522_ConfigISOType('A');
}
```
